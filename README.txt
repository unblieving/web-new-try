2. Docker Compose 启动命令
------------------------------------------------------------
在项目根目录下执行：

    docker compose -f infra/compose.yaml up --build -d

启动后：
  - 前端（Next.js）：http://localhost:3000
  - 后端（Midway.js）：http://localhost:7001

停止服务：

    docker compose -f infra/compose.yaml down

如需清除数据卷（重置数据库）：

    docker compose -f infra/compose.yaml down -v


3. 数据库和资源文件挂载说明
------------------------------------------------------------
- 数据库：SQLite，文件路径为容器内 /app/backend/data/course-demo.sqlite
- Docker Compose 使用命名卷 course-data 挂载到 /app/backend/data，
  数据在容器重启后持久保留。
- 首次启动时，后端自动建表并填充种子数据（分类、演示用户、示例商品），
  无需手动导入 SQL 或初始化数据库。
- 如需备份数据库，可从容器中拷贝：

    docker cp <container_name>:/app/backend/data/course-demo.sqlite ./backup.sqlite

5. 课程中印象最深刻的内容（可选）
------------------------------------------------------------
印象最深刻的是 SQLite 并发控制与竞态条件的处理。通过实践理解了
BEGIN IMMEDIATE 事务在 WAL 模式下的写锁机制，以及如何在应用层
通过事务隔离来防止超卖问题。从最初"先查后写"导致的竞态漏洞，
到最终用即时事务 + 事务内二次校验实现原子购买，这个过程让理论
知识变得非常具体。
